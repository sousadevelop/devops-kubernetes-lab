# Roadmap

## Current Phase

The repository is a collection of Kubernetes study manifests for workloads, networking, storage, probes, scaling, and metrics.

## Next Steps

- Group manifests by learning stage after confirming the intended exercise sequence.
- Create sanitized manifest templates with placeholders.
- Replace ConfigMap credentials with Secret examples.
- Review HPA API compatibility and target references.
- Review StatefulSet configuration references.
- Update metrics server examples for the target Kubernetes version.
- Add validation scripts for YAML and server-side dry runs.
- Add sanitized diagrams and screenshots for each exercise.

## Known Limitations

- Manifests are currently stored at repository root.
- Some files are alternative exercise versions, not one deployable bundle.
- Hardcoded lab values require review before reuse.
- Cluster compatibility has not been certified.
